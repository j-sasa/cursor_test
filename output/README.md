# タスク分析ダッシュボード

日報データから自動的にタスクを分析し、視覚的に表示するWebアプリケーションです。

## 📊 機能

### 統計サマリー
- 分析期間（日数）
- 総業務時間
- 平均業務時間
- 平均評価

### グラフ表示
- **業務時間の推移**: 日別の業務時間を折れ線グラフで表示
- **業務種別の時間配分**: カテゴリ別の時間配分を円グラフで表示
- **総合評価の推移**: 日別の評価を折れ線グラフで表示
- **日別タスク数と完了状況**: 完了・未完了タスクを棒グラフで表示

### リスト表示
- 主要な成果・達成事項の一覧
- 課題・問題点の一覧
- 全タスク詳細テーブル

## 🚀 使い方

### 方法1: HTTPサーバーを使用（推奨）

1. コマンドプロンプトまたはPowerShellで以下のコマンドを実行：

```bash
cd output
python -m http.server 8080
```

2. ブラウザで以下のURLにアクセス：

```
http://localhost:8080/task-analysis.html
```

### 方法2: 直接ファイルを開く

`task-analysis.html` をダブルクリックしてブラウザで開くこともできますが、一部のブラウザではセキュリティ制限により日報ファイルの読み込みに失敗する可能性があります。その場合は方法1をお試しください。

## 🔧 技術仕様

- **HTML5**: ページ構造
- **CSS3**: スタイリング、レスポンシブデザイン
- **JavaScript (ES6+)**: データ解析とインタラクティブ機能
- **Chart.js**: グラフ描画ライブラリ（CDN経由）

## 📝 日報データの追加

新しい日報を追加する場合：

1. `YYYY-MM-DD_日報.md` の形式でファイルを作成
2. 既存の日報と同じフォーマットで内容を記載
3. `js/analyzer.js` の `findReportFiles()` メソッド内の `reportFiles` 配列に新しいファイル名を追加

```javascript
this.reportFiles = [
    '2025-11-03_日報.md',
    '2025-11-04_日報.md',
    // ... 既存のファイル
    '2025-11-09_日報.md',  // 新しいファイルを追加
];
```

## 🎨 カスタマイズ

### カラースキームの変更

`css/analysis-styles.css` の `:root` セクションで色を変更できます：

```css
:root {
    --primary-color: #2563eb;      /* メインカラー */
    --secondary-color: #10b981;    /* サブカラー */
    --warning-color: #f59e0b;      /* 警告色 */
    --danger-color: #ef4444;       /* 危険色 */
}
```

### タスクカテゴリの変更

`js/analyzer.js` の `categorizeTask()` メソッドでカテゴリを追加・変更できます。

## 🐛 トラブルシューティング

### データが表示されない場合

1. ブラウザの開発者ツール（F12）を開いてコンソールを確認
2. HTTPサーバーが起動しているか確認
3. 日報ファイルが正しい場所にあるか確認
4. 日報ファイルのフォーマットが正しいか確認

### グラフが表示されない場合

- インターネット接続を確認（Chart.jsをCDNから読み込むため）
- ブラウザのキャッシュをクリアして再読み込み
