# タスク分析ダッシュボード - 使い方ガイド

## 🎯 概要

このダッシュボードは、outputフォルダ内の日報ファイルを自動的に解析し、以下の分析結果を視覚的に表示します：

- 業務時間の推移
- 業務種別の時間配分
- タスクの完了状況
- 成果と課題の一覧
- 総合的な統計情報

## 📖 起動方法

### ステップ1: HTTPサーバーの起動

PowerShellまたはコマンドプロンプトで以下を実行：

```powershell
cd output
python -m http.server 8080
```

サーバーが起動すると、以下のメッセージが表示されます：
```
Serving HTTP on :: port 8080 (http://[::]:8080/) ...
```

### ステップ2: ブラウザでアクセス

ブラウザで以下のURLを開きます：

```
http://localhost:8080/task-analysis.html
```

または、以下のコマンドで自動的にブラウザが開きます：

```powershell
Start-Process "http://localhost:8080/task-analysis.html"
```

## 📊 ダッシュボードの見方

### 1. 統計サマリー（上部）

4つのカードで主要な指標を表示：

- **分析期間**: 解析した日報の日数
- **総業務時間**: 期間中の合計作業時間
- **平均業務時間**: 1日あたりの平均作業時間
- **平均評価**: 日報の総合評価の平均値

### 2. グラフエリア

#### 業務時間の推移（折れ線グラフ）
- 横軸: 日付
- 縦軸: 業務時間（時間）
- 各日の業務時間の変化が一目でわかります

#### 業務種別の時間配分（円グラフ）
- 業務をカテゴリ別に分類
- 各カテゴリの時間配分を％で表示
- カテゴリ例：
  - 会議・ミーティング
  - 顧客対応
  - 資料作成
  - 企画・検討
  - 講座・教育
  - その他

#### 総合評価の推移（折れ線グラフ）
- 各日の自己評価（⭐1〜5）の推移
- 業務の充実度や満足度の変化を把握

#### 日別タスク数と完了状況（棒グラフ）
- 緑色: 完了したタスク
- オレンジ色: 未完了のタスク
- 各日のタスク量と完了率を視覚化

### 3. 成果・達成事項リスト

- 日付ごとの主要な成果を一覧表示
- 直近10件を表示
- 自分の業績を振り返る際に便利

### 4. 課題・問題点リスト

- 日付ごとの課題や問題点を一覧表示
- 繰り返し発生する課題の発見に役立つ
- 改善のための気づきを得られる

### 5. 全タスク一覧テーブル

すべてのタスクを詳細に表示：

- **日付**: タスクの実施日
- **業務内容**: タスクの名称
- **カテゴリ**: 業務の種類
- **所要時間**: かかった時間
- **進捗状況**: 完了/未完了/進捗率

## 🔄 日報データの追加・更新

### 新しい日報を追加する手順

1. **日報ファイルの作成**
   
   `output/` フォルダに `YYYY-MM-DD_日報.md` 形式でファイルを作成します。
   
   例: `2025-11-09_日報.md`

2. **analyzer.jsの更新**
   
   `output/js/analyzer.js` を開き、`findReportFiles()` メソッド内の配列に新しいファイル名を追加：

   ```javascript
   this.reportFiles = [
       '2025-11-03_日報.md',
       '2025-11-04_日報.md',
       '2025-11-05_日報.md',
       '2025-11-06_日報.md',
       '2025-11-07_日報.md',
       '2025-11-08_日報.md',
       '2025-11-09_日報.md',  // ← 追加
   ];
   ```

3. **ブラウザで再読み込み**
   
   ブラウザで `Ctrl + F5`（強制再読み込み）を実行して、新しいデータを反映させます。

- 時間がかかっているカテゴリを特定

## 🐛 トラブルシューティング

### データが表示されない

**症状**: ページは開くがグラフやデータが表示されない

**対処法**:
1. ブラウザの開発者ツール（F12）を開く
2. コンソールタブでエラーメッセージを確認
3. よくあるエラー：
   - `Failed to fetch`: HTTPサーバーが起動していない
   - `404 Not Found`: ファイルパスが間違っている

### グラフが崩れる

**症状**: グラフの表示がおかしい

**対処法**:
1. ブラウザのキャッシュをクリア
2. `Ctrl + F5` で強制再読み込み
3. インターネット接続を確認（Chart.jsをCDNから読み込むため）

### サーバーが起動しない

**症状**: `python -m http.server 8080` でエラーが出る

**対処法**:
1. Pythonがインストールされているか確認
2. ポート8080が既に使用されている場合は別のポートを使用：
   ```powershell
   python -m http.server 8081
   ```
   その場合、URLも `http://localhost:8081/task-analysis.html` に変更

## 📝 日報のフォーマット要件

ダッシュボードが正しく動作するために、日報は以下のフォーマットに従う必要があります：

### 必須セクション

```markdown
# 日報

## 基本情報
- **日付**: YYYY年MM月DD日（曜日）

## 本日の業務内容
### 業務1: タスク名
- **内容**: ...
- **所要時間**: 約X時間
- **進捗状況**: 完了

## 本日の成果・達成事項
- 成果1
- 成果2

## 課題・問題点
- **課題1**: 課題の概要

## 総括
**本日の業務時間**: X時間
**総合評価**: ⭐⭐⭐⭐⭐ (5段階)
```
